PROJECT: A/B Testing

In this project, I did A/B testing with 2 versions of an online cactus shopping website.
I collected data to test whether there's statistically significant differences in time to completion and return rate between the 2 versions.

Website Design

We first create 2 versions of the same website.
This is version A of the website:

I've made some changes to the interface and created version B:

Compared to version A, the version B has a few differences:

  • I changed the text on the button from '# in cart' to 'Add to cart'.
  • I moved the position of price to be after the product descriptions.
  • I changed the text next to the buttons to display the number of product in cart.

My Hypotheses

I have made null hypothesis and alternative hypothesis for each metric we study:

Metric: Time to completion

Time to complete is the time a use takes to finish the workflow.

  • Null hypothesis: The time to completion for version A is equal to the time to completion for version B.
  • Alternative hypothesis: The time to completion for version B is shorter than the time to completion for version A, because the 'Add to cart' buttons in B are more prominent as the button text indicates the action more clearly.

Metric: Return rate

Return rate is the percentage of sessions where users return to the workflow after going to the check out page.

  • Null hypothesis: The return rate for version A is equal to the return rate for version B.
  • Alternative hypothesis: The return rate for version B is higher than the return rate for version A, because prices are not as prominent, since users goal is to add 150 dollar worth of purchase to cart, they will likely return more if they can't clearly the prices.

Data Collection & Analysis

After I made the 2 versions of the website. I collect data through the studio, then I cleaned the user session logs. For my analysis, I used Excel to calculate time to completion and record whether the user returned for each session from the logs.
I calculated time to completion by subtracting the timestamp of end of session from the timestamp of start of session. And I determined whether users returned by looking at if they return to page A/B after visiting page C.

Metric: Time to completion

  • Version A: The mean is 77488.6.
  • Version B: The mean is 53680.1.

I used t-test to measure if there is a statistically significant difference, after calculation, t-score is -0.619 and p-value is 0.271.
Because p-value is larger than 0.05, we fail to reject null hypothesis, the difference of time to completion between 2 versions is not statistically significant.

Metric: Return rate

  • Version A: Out of 15 sessions, users return in 9 sessions, not return in 6 sessions. Return rate is 0.600.
  • Version B: Out of 9 sessions, users return in 7 sessions, not return in 2 sessions. Return rate is 0.778.

I used chi-square test to measure if there is a statistically significant difference, after calculation, chi-square score is 0.800 and p-value is 0.371.
Because p-value is larger than 0.05, we fail to reject null hypothesis, the difference of return rate between 2 versions is not statistically significant.

Results & Reflections

Version B shows shorter time to completion and higher return rate than version A, however neither of the test result are statistically significant.
The placement of prices on version B makes it hard for advertisers to read, thus they return more when they add items to the cart. The buttons on version B are more indicative of their functionalities and users are likely to make decisions to add items more quickly.

I've also created an infographic that describes all the results and conclusions from each statistical tests.

My takeways on A/B testing from this project are:

  • A/B testing is a useful technique when we want to quantitatively measure differences in user's behaviors on two interfaces. It also has a lot of real-world aaplications.
  • In A/B testing, it's important to focus on the important metrics we are interested in (in this study, we focused on time to completion and return rate) and use correct methods to measure whether there are statistically signifcant differences between metrics of 2 interfaces.
  • Larger sample size will help detect the significance of the differences more accurately.
  • Even small interface changes can lead to changes in users' behavior, and we can use data from these studies to support our design decisions.
